# Sidebar Resize Handle Hover Fix

## Issue Description
The sidebar resize handle's visual indicator (the vertical line) was appearing incorrectly.
- **Expected Behavior**: The line should only appear when hovering over the resize handle area (the 16px strip where the cursor turns to `<->`).
- **Actual Behavior**: The line was appearing when hovering *anywhere* on the sidebar or when the parent group was hovered, due to CSS group nesting conflicts.

## Root Cause
The `Sidebar` component uses the Tailwind `group` class on parent containers for various styling states.
The resize handle was also using the standard `group` class:
```tsx
<div className="... group">
  <div className="... group-hover:bg-primary/50" />
</div>
```
Because the sidebar itself (or a parent wrapper) also had `group`, the `group-hover` on the resize line was being triggered by the parent's hover state, causing the line to appear whenever the sidebar was hovered.

## Solution: Named Groups
To fix this, we used Tailwind's **Named Groups** feature to isolate the hover context.

### Implementation
1.  Assigned a specific name to the resize handle's group: `group/resize`.
2.  Updated the inner line to respond only to that specific group: `group-hover/resize:...`.

### Code Change
**File:** `components/ui/sidebar.tsx`

```tsx
// BEFORE
<div
  className="absolute right-0 top-0 bottom-0 w-4 cursor-ew-resize z-30 -mr-2 group" // Generic group
  onMouseDown={() => setIsDragging(true)}
>
  <div className="absolute right-2 top-0 bottom-0 w-1 bg-transparent group-hover:bg-primary/50 transition-colors" />
</div>

// AFTER
<div
  className="absolute right-0 top-0 bottom-0 w-4 cursor-ew-resize z-30 -mr-2 group/resize" // Named group
  onMouseDown={() => setIsDragging(true)}
>
  <div className="absolute right-2 top-0 bottom-0 w-1 bg-transparent group-hover/resize:bg-primary/50 transition-colors" />
</div>
```

## Key Takeaway
When working with nested components that both rely on `group-hover` logic, always use **Named Groups** (`group/{name}` and `group-hover/{name}`) to prevent parent hover states from accidentally triggering child styles.
