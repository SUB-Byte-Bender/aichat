# Mobile Code Block Overflow Fix

## Problem

On mobile devices, code blocks inside chat message bubbles were overflowing beyond the bubble container, breaking the layout. The code block would extend past the visible screen area instead of staying contained within the message bubble.

## Root Cause

The issue stemmed from CSS flex containers not properly constraining their children's width. Even with `max-w-[80%]` on the container and `overflow-x-auto` on the `<pre>` element, the content would push the container wider because:

1. Flex children with content (like code blocks) can grow beyond their container
2. `max-width` percentages don't always propagate correctly through nested flex containers
3. The `prose` class from Tailwind Typography uses `max-w-none` which removes width constraints

## Solution

Used **CSS Grid with `minmax(0, 1fr)`** - a well-known technique to properly constrain content in grid/flex layouts.

### Changes Made in `components/chat/message-bubble.tsx`

#### 1. Changed the bubble container from flex to grid:

```tsx
// Before
<div ref={containerRef} className="relative flex max-w-[80%] min-w-0 flex-col">

// After
<div ref={containerRef} className="relative grid max-w-[80%] min-w-0" style={{ gridTemplateColumns: 'minmax(0, 1fr)' }}>
```

#### 2. Added `max-w-full` to the `<pre>` element:

```tsx
// Before
<pre className={cn("bg-zinc-900 text-zinc-50 dark:bg-zinc-950 overflow-x-auto", ...)}>

// After  
<pre className={cn("bg-zinc-900 text-zinc-50 dark:bg-zinc-950 overflow-x-auto max-w-full", ...)}>
```

## Why This Works

### The `minmax(0, 1fr)` Trick

- `1fr` means the column takes up all available space
- `minmax(0, 1fr)` sets the **minimum** width to `0` instead of `auto`
- With `min-width: auto` (the default), grid/flex items won't shrink below their content size
- With `min-width: 0` (via `minmax(0, ...)`), the item CAN shrink below content size
- This allows `overflow-x-auto` to actually work, creating a scrollable area

### Why Previous Approaches Failed

| Approach | Why It Failed |
|----------|---------------|
| `overflow-hidden` on parent | Clips content but doesn't create scroll for code |
| `max-w-full` alone | Doesn't work when parent has `min-width: auto` |
| `calc(100vw - Xpx)` | Not dynamic across all screen sizes |
| JavaScript width measurement | Over-engineered, timing issues |

## Result

- Code blocks now stay contained within the message bubble at all screen sizes
- Long code lines scroll horizontally within the code block
- No fixed pixel calculations needed - fully responsive
- Works on mobile, tablet, and desktop
