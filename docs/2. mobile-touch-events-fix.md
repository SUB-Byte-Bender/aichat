# Fixing Mobile Double-Tap Issues

## Problem
Buttons in the app require double-tapping on mobile devices to work, causing poor user experience.

## Root Cause
Mobile browsers have a delay on `onClick` events (typically 300ms) to detect double-tap-to-zoom gestures. When a button has hover states or certain CSS properties, the first tap often triggers the hover state, and the second tap triggers the actual click.

## Solution: Use `onTouchEnd`

### Implementation
Add an `onTouchEnd` handler alongside `onClick` to ensure immediate response on mobile devices:

```tsx
function MyButton() {
    const [copied, setCopied] = useState(false);

    const handleAction = async () => {
        // Your main action here
        setCopied(true);
        setTimeout(() => setCopied(false), 2000);
    };

    // Add this handler for mobile
    const handleTouch = (e: React.TouchEvent) => {
        e.preventDefault(); // Prevents the 300ms delay
        handleAction();
    };

    return (
        <button
            onTouchEnd={handleTouch}  // For mobile - instant response
            onClick={handleAction}     // For desktop - fallback
            className="..."
        >
            Click Me
        </button>
    );
}
```

### Key Points

1. **`onTouchEnd` fires immediately** when the user lifts their finger on a touch device
2. **`e.preventDefault()`** is crucial - it prevents the delayed click event from firing
3. **Keep `onClick`** as a fallback for non-touch devices (desktop, stylus, etc.)
4. **Both handlers call the same function** to maintain consistent behavior

### What NOT to Do (Tried But Failed)

❌ **Using `touch-action: manipulation`** - CSS property alone doesn't fix the issue in all browsers

❌ **Using `@media(hover:hover)`** - Only hides hover styles, doesn't fix the delay

❌ **Removing `title` attribute** - Doesn't address the core onClick delay

❌ **Adding `active` states** - Visual feedback but doesn't solve the delay

❌ **Increasing button size or z-index** - Doesn't affect touch event timing

### When to Use This Pattern

Apply this fix when:
- Buttons require instant feedback (copy buttons, toggle buttons, etc.)
- Users report needing to tap twice on mobile
- Buttons have hover states or transitions
- Critical user actions need immediate response

### Example: Code Copy Button

```tsx
// Code Copy Button Component
function CodeCopyButton({ code }: { code: string }) {
    const [copied, setCopied] = useState(false);

    const handleCopy = async () => {
        await navigator.clipboard.writeText(code);
        setCopied(true);
        setTimeout(() => setCopied(false), 2000);
    };

    const handleTouch = (e: React.TouchEvent) => {
        e.preventDefault();
        handleCopy();
    };

    return (
        <button
            onTouchEnd={handleTouch}
            onClick={handleCopy}
            className="absolute left-0 top-0 bottom-0 w-6 bg-zinc-800 hover:bg-zinc-700 transition-colors flex items-center justify-center text-[10px] font-medium tracking-wider border-r border-zinc-700 hover:text-white"
            style={{ writingMode: 'vertical-rl', textOrientation: 'mixed', transform: 'rotate(180deg)', color: '#bababa' }}
            title='Copy code'
        >
            {copied ? <span style={{ transform: 'rotate(-180deg)', display: 'inline-block' }}>✓</span> : 'COPY CODE'}
        </button>
    );
}
```

## Testing
After implementing:
1. Test on actual mobile device (not just browser dev tools)
2. Verify single tap triggers the action
3. Verify desktop clicking still works
4. Check that double-tap-to-zoom is still disabled where needed
